2.1. UQ: Uncertainty Quantification¶

The first selection panel the user must select from and enter data into is the UQ tab. It is in this panel where the user selects the UQ Engine to use for performing the uncertainty quantification calculations.

Presently the UQ Engine option supports Dakota, SimCenterUQ, UCSD-UQ and users’ own custom UQ engines.

2.1.1. Dakota UQ Engine¶

This UQ engine utilizes the Dakota Software, a state-of-the-art research application that is robust and provides many methods for optimization and UQ, a selection of which we utilize in this application. Dakota provides the user with a large number of different methods. For this reason we have divided the methods into categories though a pull-down menu, as shown in Fig. 2.1.1.1. Once the category has been selected, a number of different methods are made available to the user.

Fig. 2.1.1.1 Dakota engine and category selection.¶

The following categories are available:

  • 2.1.1.1. Forward Propagation Methods
  • 2.1.1.2. Global Sensitivity Analysis
  • 2.1.1.3. Reliability Analysis
  • 2.1.1.4. Parameter Estimation
  • 2.1.1.5. Inverse Problem

2.1.2. SimCenter UQ Engine¶

The SimCenterUQ engine is a UQ engine developed in-house at the SimCenter that accommodates different UQ methods, which are organized into categories that can be accessed through a pull-down menu, as shown in Fig. 2.1.2.1.

Fig. 2.1.2.1 SimCenterUQ engine and category selection.¶

Currently the category option is limited to global sensitivity analysis:

  • 2.1.2.1. Forward Propagation Methods
  • 2.1.2.2. Global Sensitivity Analysis
  • 2.1.2.3. Surrogate Modeling

2.1.3. UCSD UQ Engine¶

The UCSD-UQ engine is a module developed at the SimCenter in collaboration with UCSD. It provides algrithms for Bayesian estimation, which can be accessed through a pull-down menu, as shown in figure Fig. 2.1.3.1.

Fig. 2.1.3.1 UCSD-UQ engine and category selection.¶

This module currently offers support for Bayesian estimation using the Transitional Markov chain Monte Carlo (TMCMC) algorithm, which is also known as the Sequential Monte Carlo algorithm:

  • 2.1.3.1. Transitional Markov chain Monte Carlo (TMCMC) algorithm

2.1.4. Custom UQ Engine¶

The CustomUQ option enables users to switch out the UQ engine in the quoFEM workflow such that different methods and tools can be applied within the SimCenter framework with minimal effort on the part of the user. The CustomUQ option can be accessed as shown in figure Fig. 2.1.4.1.

Fig. 2.1.4.1 CustomUQ engine selection.¶

In order to use the CustomUQ engine option, two steps are required:

  • Configuring UQ tab to accept the required inputs

  • Adding UQ engine to customized UQ backend

These steps are described in more detail here:

  • 2.1.4.1. Configuring CustomUQ Engine
